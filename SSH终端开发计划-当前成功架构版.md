# Windows SSH终端开发计划（当前成功架构版）

## 1. 当前已验证成功的技术架构

本版本已跑通“连接 -> 输入密码 -> 执行命令 -> 返回输出”完整链路，当前采用架构如下：

- 桌面框架：`Tauri 2.0`（已完成）
- 后端语言：`Rust 1.77+`（已完成）
- 终端进程管理：`portable-pty 0.8`（已完成）
- SSH实现：调用系统自带 `ssh.exe`（不使用第三方SSH协议库）（已完成）
- 前端终端组件：`xterm.js 5.3` + `xterm-addon-fit`（已完成）

### 数据与交互流

1. 前端输入地址 `user@host:port`，点击“连接”（已完成）。
2. 前端调用 Rust 命令 `connect_ssh`（已完成）。
3. Rust 通过 `portable-pty` 创建 PTY，在 PTY 内启动 `ssh.exe`（已完成）。
4. PTY 输出通过 Tauri 事件 `ssh-output` 推送到前端（已完成）。
5. 前端 `xterm.js` 将用户输入通过 `send_ssh_input` 发送给 PTY（已完成）。
6. 窗口/终端尺寸变化时，前端调用 `resize_ssh` 同步 PTY 行列（已完成）。

## 3. 后续开发计划（建议按阶段执行）

## 阶段A：稳定性增强（1-2周）

目标：先把“能用”提升到“稳定可用”。

- 增加连接状态机（idle/connecting/authenticated/disconnected）
- 增加连接超时与明确错误分类（DNS失败、端口不可达、认证失败）
- 规范日志级别（info/warn/error）与日志导出
- 补充断线回收机制，避免僵尸会话

验收标准：

- 连续连接/断开 200 次无崩溃
- 异常场景均能在UI看到可理解错误信息

## 阶段B：可用性增强（2-3周）

目标：接近 Xshell 的日常体验。

- 地址历史与收藏（最近连接、常用连接）
- 会话标签页（多连接并行）
- 快捷键扩展（Ctrl+Insert / Shift+Insert / Ctrl+Shift+C/V）
- 终端字体、主题、滚动缓冲配置

验收标准：

- 多会话并行稳定，输入输出不串流
- 常用连接可一键重连

## 阶段C：工程化与发布（1-2周）

目标：可分发、可回归、可持续迭代。

- 增加端到端冒烟脚本（连接、执行 `ls`、断开）
- 打包流程固化（debug/release）
- 崩溃与错误采集最小闭环
- 发布说明模板与版本号规范

验收标准：

- 新机器可按文档 15 分钟内跑起并连接成功
- 每次发布均有可回滚版本与变更记录

## 4. 架构演进路线（保持纯系统ssh.exe路线）

为满足性能与长期维护目标，后续坚持以下原则：

- SSH协议层继续使用系统 `ssh.exe`
- 终端交互层继续使用 `portable-pty`
- 前端渲染层继续使用 `xterm.js`
- 避免引入第三方SSH协议实现库，降低协议维护风险

中长期可选增强：

- 连接复用与会话池
- 配置加密存储（凭据不落明文）
- SFTP子系统（仍可复用系统能力或独立模块）
